# 在 VSCode 中创建并发布 Hugo 博客文章的完整流程

以下步骤假设你已经配置好 Hugo 环境，且项目位于 `D:\code\lokasi.github.io`。我们将从创建文章开始，一直到将更新部署到 GitHub Pages。

---

## 1. 在 VSCode 中创建新文章

有两种方式：使用终端命令，或手动创建文件。

### ✅ 方式一：使用 Hugo 命令（推荐）
1. 打开 VSCode，按 `Ctrl+`` 打开集成终端。
2. 确保当前目录是你的博客根目录（例如 `D:\code\lokasi.github.io`）。
3. 运行以下命令创建新文章（将 `my-new-article` 替换为你想要的文件名，建议英文小写和连字符）：
   ```bash
   hugo new posts/my-new-article.md
   ```
   Hugo 会自动在 `content/posts/` 下生成文件，并填充默认的 Front Matter。

### ✅ 方式二：手动创建文件
1. 在 VSCode 的资源管理器中，导航到 `content/posts/` 文件夹。
2. 右键选择“新建文件”，命名为 `my-new-article.md`（注意扩展名 `.md`）。
3. 打开文件，复制以下内容作为模板（根据你使用的 Front Matter 格式调整）：
   ```toml
   +++
   title = '我的新文章'
   date = 2026-03-08T20:30:00+08:00
   draft = true
   +++
   ```
   如果你喜欢 YAML 格式，可以用：
   ```yaml
   ---
   title: "我的新文章"
   date: 2026-03-08T20:30:00+08:00
   draft: true
   ---
   ```

---

## 2. 编辑文章内容

1. 在 VSCode 中打开刚才创建的文件（位于 `content/posts/`）。
2. **修改 Front Matter**：
   - 将 `title` 改为你文章的标题。
   - 日期通常保持自动生成即可，也可手动调整。
   - 如果文章还没写完，保持 `draft: true`；写完准备发布时改为 `draft: false`。
3. **在 Front Matter 下方写正文**，使用 Markdown 语法。例如：
   ```markdown
   # 标题

   这是正文。**支持加粗**，也可以插入[链接](https://example.com)。

   - 列表项1
   - 列表项2

   ![图片描述](/images/图片名.jpg)
   ```

**插入图片**：将图片文件放入 `static/images/` 目录，然后在文中用 `![描述](/images/图片名.jpg)` 引用。

---

## 3. 本地预览

在 VSCode 终端中运行：
```bash
hugo server
```
如果文章还是草稿（`draft: true`），需要加上 `-D` 参数才能预览：
```bash
hugo server -D
```

打开浏览器访问 `http://localhost:1313/lokasi.github.io/`，查看文章效果。Hugo 支持热加载，修改文件后保存，浏览器会自动刷新。

---

## 4. 确认文章可发布

- 将文章文件中的 `draft` 改为 `false`（如果还没改）。
- 再次本地预览，确认文章已显示且格式正确。

---

## 5. 生成静态文件

在终端运行：
```bash
hugo --cleanDestinationDir
```
这会清空 `public/` 目录并重新生成所有静态文件（HTML、CSS、JS 等）。

---

## 6. 将新文件部署到 GitHub Pages（手动部署）

### 6.1 删除根目录下旧的生成文件
执行以下命令（避免旧文件干扰）：
```powershell
Remove-Item -Recurse -Force css, js, fonts, categories, page, tags, index.html, index.xml, sitemap.xml -ErrorAction SilentlyContinue
```

### 6.2 复制新生成的 `public/` 内容到根目录
```powershell
Copy-Item -Recurse -Force public\* .
```

### 6.3 提交更改到 Git
```powershell
git add .
git commit -m "发布新文章：我的新文章"
```

### 6.4 推送到 GitHub
```powershell
git push origin main
```
如果网络需要代理，确保代理已开启（你之前配置的是 SOCKS5 10808 端口，Git 已设置好）。

---

## 7. 等待部署并查看线上网站

推送成功后，GitHub Pages 会自动更新，通常需要 1-2 分钟。之后在浏览器中访问：
```
https://taloy6.github.io/lokasi.github.io/
```
就能看到新文章出现在首页。

---

## 8. 后续更新

- 每次写完新文章，重复 **步骤 4 → 步骤 5 → 步骤 6** 即可。
- 如果修改了主题配置或模板，同样需要重新生成并推送。

## 常见问题

### ❓ 为什么 `git push` 后网站没更新？
- 检查 GitHub Actions 是否运行（如果你配置了自动部署）。手动部署的话，确认你复制了 `public` 内容并提交推送。
- 检查浏览器缓存，按 `Ctrl+F5` 强制刷新。
- 等待 1-2 分钟，GitHub Pages 可能需要时间。

### ❓ 文章不显示？
- 确认 `draft: false`。
- 确认文件在 `content/posts/` 下，且文件名没有特殊字符（建议只使用字母、数字、连字符）。
- 本地预览正常吗？如果本地也看不到，可能是 front matter 格式错误。

### ❓ 图片显示 404？
- 确认图片放在 `static/images/` 下。
- 引用路径以 `/images/` 开头，例如 `![猫](/images/cat.jpg)`。
- 检查文件名大小写和扩展名。
